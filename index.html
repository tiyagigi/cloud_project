Project Architecture

Detailed breakdown of our systems and deployments.

Static Portfolio Site (GitOps)

  • App: Simple HTML/Tailwind portfolio (This site)
  • Infra: Terraform Route53 pointing to AWS EKS cluster
  • CI/CD: GitHub Actions comments `terraform plan` on PRs
  • Deploy: Nginx serves files with HSTS and CSP Security headers
  • Ops: Prometheus monitoring for 404 errors

Global Content Delivery System

  • Infra: Terraform provisions CloudFront distribution with S3 origin
  • Local Nginx: Local Nginx cache configured on WSL2 imitating a CDN
  • Automation: GitHub action handles cache invalidation on EKS image push via AWS CLI
  • Goal: Ensure scalability and high-availability edge optimization worldwide
